        Usage Guide

        This form is used to add new colors to your palette.

        The Input Methods provide different notations for colors and might employ different colorspaces (see the detailled description below). Most of the notations rely on different components, usually mapping to coordinates in a colorspace or are translatable into coordinates.

        Each component might be adjusted using a slider or a direct, text-based input field.

        Input Methods

        Colorizer supports the following notations for input of colors:

        RGB

        The RGB notation is the most basic method to specify a color in sRGB colorspace.

        It represents a color by specifying the Red, Green, and Blue components of the color with values between 0 and 255.

        HSL

        The HSL notation is an alternate method to specify a color in sRGB colorspace.

        The Hue of the color is specified in degree in a range between 0 and 360. The Saturation and Lightness of the color are specified in percent between 0 and 100.

        For Lightness values of 0% and 100% the values of hue and saturation are considered powerless, meaning they have no effect on the actual color, which will be black / white.

        Please note that the Lightness of the HSL notation is just a linear and artificial value. This means, that hsl(120 100% 50%) and hsl(240 100% 50%) share the lightness value of 50%, but the green seems significantly brighter than the blue. The reason for this is, that the perceived lightness of green and blue hues is different. Other input methods (e.g. OkLCH) do provide the direct manipluation of perceived lightness.

        HWB

        HWB is another alternate method to specify colors in sRGB colorspace.

        The Hue of the color is specified in degree in a range between 0 and 360. Whiteness and Blackness specify the amount of white/black mixed into the color and are specified in percent between 0 and 100.

        OkLCH

        OkLCH is the polar notation for the OkLAB colorspace.

        The Lightness component is the perceived lightness of the color, specified in percent in range 0 and 100. The Chroma component specifies how colorful the color is. Colorizer provides inputs based on percent. However, the value range is not really clipped! The Hue component is specified in degree in a range between 0 and 360 (please note that this Hue is not synchronized with HSL's or HWB's hue components!).
